Паттер "Фабричный метод" (так же называется "Виртуальный конструктор") предназначен для того, чтобы отложить создание
конкретного экземпляра класса. В этом примере если бы не было фабричного метода, то в оружейне нам приходилось бы создавать
каждое оружние отдельно. Это привело бы к, например, увеличению числа методов до числа видов оружия:
    createBow();
    createSword();
    createAxe();
А так у нас будет 1 метод - createWeapon() который будет создавать экзепляры класса Weapon, конкретная реализация которого
будет определяться позже
